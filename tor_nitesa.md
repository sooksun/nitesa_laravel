# รายละเอียดการจ้างงาน (TOR)
## ระบบนิเทศ ติดตาม และประเมินผลการศึกษา (NITESA)

---

## 1. หลักการและเหตุผล

การนิเทศ ติดตาม และประเมินผลการศึกษาเป็นภารกิจสำคัญของสำนักงานเขตพื้นที่การศึกษา ในการส่งเสริมและพัฒนาคุณภาพการจัดการศึกษาของสถานศึกษาในสังกัด ให้มีการจัดการเรียนรู้ที่มีประสิทธิภาพ และผู้เรียนได้รับการพัฒนาตามศักยภาพ

ปัจจุบันการดำเนินงานนิเทศการศึกษาของหลายเขตพื้นที่ยังอาศัยการบันทึกแบบเอกสารหรือระบบที่แยกส่วน ทำให้การติดตามผล การสรุปข้อมูล และการรายงานเป็นไปด้วยความยากลำบาก และไม่เอื้อต่อการวิเคราะห์ภาพรวมในระดับเขตพื้นที่หรือระดับนโยบาย

เพื่อให้การนิเทศ ติดตาม และประเมินผลการศึกษามีความต่อเนื่อง เป็นระบบ และสามารถนำข้อมูลไปใช้ในการวางแผนและพัฒนาคุณภาพการศึกษาได้อย่างมีประสิทธิภาพ จึงจำเป็นต้องมีระบบสารสนเทศที่รองรับการบันทึกการนิเทศ การจัดการข้อมูลโรงเรียนและนโยบาย การดำเนินงานตาม Workflow การอนุมัติหลายระดับ รวมถึงการรายงานเชิงวิเคราะห์ในรูปแบบ Dashboard รายงาน และกราฟ

ระบบนิเทศ ติดตาม และประเมินผลการศึกษา (NITESA) จึงถูกพัฒนาขึ้นเป็น Web Application เพื่อใช้เป็นเครื่องมือในการนิเทศ ติดตาม และประเมินผลการศึกษาของสำนักงานเขตพื้นที่การศึกษา ให้สามารถบริหารจัดการข้อมูลโรงเรียน นโยบาย การบันทึกการนิเทศ และรายงานผลได้อย่างเป็นระบบและครบวงจร

---

## 2. เป้าประสงค์

เพื่อพัฒนาและจัดส่งระบบนิเทศ ติดตาม และประเมินผลการศึกษา (NITESA) ที่มีความสามารถในการใช้เป็นเครื่องมือในการนิเทศ ติดตาม และประเมินผลการศึกษาของสำนักงานเขตพื้นที่การศึกษา พร้อมคู่มือการใช้งานและเอกสารประกอบ

---

## 3. ขอบเขต

การทำงานของระบบนี้มีลักษณะการทำงานบนเว็บเบราว์เซอร์ (Web Application) ต้องมีความสามารถในการใช้เป็นเครื่องมือในการนิเทศ ติดตาม และประเมินผลการศึกษา ดังต่อไปนี้

### 3.1 ระดับโรงเรียน (School)

3.1.1 ดูรายการการนิเทศที่เผยแพร่แล้วของโรงเรียนตนเอง  
3.1.2 ดูรายละเอียดการนิเทศ ได้แก่ ข้อมูลการนิเทศ ตัวชี้วัด คะแนน ข้อเสนอแนะ และไฟล์แนบ  
3.1.3 รับทราบผลการนิเทศ พร้อมบันทึกความคิดเห็นหรือแผนพัฒนา  
3.1.4 ดู Dashboard และรายงานที่เกี่ยวข้องกับโรงเรียนตนเอง  

### 3.2 ระดับศึกษานิเทศก์ (Supervisor)

3.2.1 ดูรายการโรงเรียนในความรับผิดชอบ  
3.2.2 บันทึก/แก้ไข และอัปโหลดข้อมูลการนิเทศ ได้แก่ วันที่นิเทศ ปีการศึกษา รายละเอียดการนิเทศ ประเด็นที่พบ ข้อเสนอแนะ ตัวชี้วัดและคะแนน ไฟล์แนบ  
3.2.3 จัดการสถานะการนิเทศ: บันทึกร่าง ส่งเพื่ออนุมัติ เผยแพร่ (เมื่ออนุมัติแล้ว)  
3.2.4 ดูรายการนโยบายและตัวชี้วัด  
3.2.5 ดู Dashboard และรายงานตามสิทธิ์  

### 3.3 ระดับผู้บริหาร (Executive)

3.3.1 ดูรายการโรงเรียนและข้อมูลพื้นฐาน  
3.3.2 ดูรายการการนิเทศที่ส่งเพื่ออนุมัติ  
3.3.3 อนุมัติหรือส่งกลับ (ต้องปรับปรุง) การนิเทศ พร้อมบันทึกเหตุผล  
3.3.4 ดู Dashboard และรายงานภาพรวม  

### 3.4 ระดับผู้ดูแลระบบ (Admin)

3.4.1 จัดการข้อมูลโรงเรียน: เพิ่ม แก้ไข ลบ ดู รายการโรงเรียน  
3.4.2 จัดการนโยบายและตัวชี้วัด: เพิ่ม แก้ไข ลบ ดู รายการนโยบาย  
3.4.3 จัดการกลุ่มเครือข่าย (Network Group)  
3.4.4 จัดการผู้ใช้งาน: เพิ่ม แก้ไข กำหนดบทบาท (Admin, Supervisor, Executive, School)  
3.4.5 ดู/อนุมัติ/ส่งกลับ การนิเทศ  
3.4.6 นำเข้าข้อมูลจากไฟล์ Excel (โรงเรียน นโยบาย กลุ่มเครือข่าย)  
3.4.7 ดู Dashboard รายงาน และส่งออกรายงาน (Excel/พิมพ์)  
3.4.8 ตั้งค่าระบบ  
3.4.9 ดู Activity Log ของระบบ  

### 3.5 ฟังก์ชันร่วม (ทุกบทบาทที่เกี่ยวข้อง)

3.5.1 Dashboard แสดงภาพรวมในมุมมองตามบทบาท (จำนวนโรงเรียน การนิเทศ สถานะ ตัวชี้วัด แนวโน้ม)  
3.5.2 รายงานในรูปแบบตาราง กราฟ และส่งออก Excel  
3.5.3 Workflow การนิเทศ: ร่าง → ส่งแล้ว → อนุมัติแล้ว → เผยแพร่แล้ว หรือ ส่งกลับ (ต้องปรับปรุง)

## 4. การส่งมอบงาน

การส่งมอบงานโปรแกรมระบบนิเทศ ติดตาม และประเมินผลการศึกษา (NITESA) ตามกรอบการทำงานในข้อ 3 พร้อมเอกสารประกอบ ดังนี้

4.1 โปรแกรมระบบ (Source Code และไฟล์ที่จำเป็นสำหรับการติดตั้ง)  
4.2 คู่มือการใช้งานระบบทั้งแบบไฟล์ PDF และในรูปแบบที่เข้าถึงได้จาก Web Application  
4.3 เอกสารการติดตั้งและรายละเอียดการ Deploy บน Server (ตามที่กำหนดในข้อ 5)  

**กำหนดส่งมอบ:** ภายในวันที่ ________ ตามที่ระบุในสัญญาจ้างหรือข้อตกลงเป็นหนังสือ  

---

## 5. เครื่อง Server และการจัดเก็บ

5.1 เครื่อง Server ที่ใช้ในการจัดเก็บฐานข้อมูลและรันแอปพลิเคชันต้องมีมาตรฐานไม่ต่ำกว่าเครื่อง Server Hosting ที่รองรับ PHP 8.1+ MySQL/MariaDB และการรัน Web Application แบบ Docker หรือ PHP-FPM กับ Nginx/Apache  

5.2 ค่าจ้างให้รวม/ไม่รวมค่าใช้จ่ายในการเช่า Web Server ตามที่ตกลงในสัญญา ผู้รับจ้าง/ผู้ว่าจ้างเป็นผู้รับผิดชอบค่าใช้จ่ายตามที่ระบุในสัญญา

## 11. อัตราค่าปรับ

หากผู้รับจ้างไม่สามารถส่งมอบงานจ้างได้ตามกำหนดเวลา หรือส่งมอบงานไม่ถูกต้องครบถ้วน หรือไม่ดำเนินการภายในกำหนดเวลาตามสัญญาหรือข้อตกลงเป็นหนังสือ และผู้ว่าจ้างยังมิได้บอกเลิกสัญญาหรือข้อตกลงเป็นหนังสือ ผู้รับจ้างจะต้องชำระค่าปรับเป็นรายวัน ในอัตราร้อยละ ________ ของราคาค่าจ้างนั้น แต่จะต้องไม่ต่ำกว่าวันละ ________ บาท นับถัดจากวันที่ครบกำหนดแล้วจนเสร็จตามสัญญาหรือข้อตกลงเป็นหนังสือ หรือวันที่ผู้ว่าจ้างได้ขยายให้จนถึงวันที่ผู้รับจ้างได้ส่งมอบงานถูกต้องครบถ้วนตามสัญญาหรือข้อตกลงเป็นหนังสือ  

นอกจากนี้ผู้รับจ้างยินยอมให้ผู้ว่าจ้างเรียกค่าเสียหายอันเกิดจากการที่ผู้รับจ้างทำงานล่าช้าเฉพาะส่วนที่เกินกว่าจำนวนค่าปรับดังกล่าวได้อีกด้วย  

*(อัตราและจำนวนตามที่กำหนดในสัญญา)*
